# 创始人手册：构建 AI 原生创业公司 — 完整结构树状图

> 来源：Anthropic / Claude 官方电子书《[The Founder's Playbook: Building an AI-Native Startup](https://claude.com/blog/the-founders-playbook)》
> 本文为全书内容结构梳理

---

## 结构总览

全书 = **背景（1-2章）** + **四阶段方法论（3-6章）** + **收尾（7章）** + **资源（8章）**

其中 3–6 章是真正的「方法论主体」，每章固定四件套，彼此高度对称：

> **目标 → 退出标准 → 挑战 → Claude 怎么帮（含练习）**

---

## 完整树状图

```
《创始人手册：构建 AI 原生创业公司》
│
├─ 第1章 创业生命周期，为 2026 重启（总论·背景）
│   ├─ 核心论断：AI 重塑创业构建方式，门槛被拉平
│   ├─ AI 的三大能力：写生产代码 / 市场研究 / 起草材料 / 自动化运营
│   └─ 传统增长曲线被打破
│       └─ 旧路径「验证→融资→招人→构建→再融资→增长」已失效
│           └─ 引出本书框架：重映四大核心阶段
│
├─ 第2章 当一名创始人的定义正在改变（角色论）
│   ├─ 创始人角色转变：从「个人执行者」→「智能体编排者」
│   ├─ 最革命性结果：解放有领域专长的非技术创始人
│   └─ 精简创业的三大 AI 能力支柱
│       ├─ ① 对话式智能与研究（随叫随到的专家）
│       │     深度研究 / 文档起草 / 战略思考伙伴
│       ├─ ② 智能体编程（永不被卡住的工程师）
│       │     用大白话指挥 AI 生成·测试·调试·重构
│       └─ ③ 工作流自动化（随需调用的运营团队）
│             排期 / CRM / 报表 / 文档 / 合规 / 系统集成
│   └─ 结论：时机与编排是一切（非自动驾驶，需创始人会用）
│
├─ 第3章 创意阶段 Idea Stage ★方法论核心
│   ├─ 目标：以研究为导向的验证（构建前先收集证据）
│   ├─ 退出标准：找到「问题—方案契合」
│   │     ① 问题真实具体？ ② 方案对准真问题？ ③ 信号足够？
│   ├─ 挑战
│   │     ├─ 把「构建」误当「验证」（原型≠证据）
│   │     ├─ 过早规模化
│   │     └─ 客观性丧失（确认偏误 + 研究引擎）
│   └─ Claude 怎么帮
│         ├─ 选对形态：Chat / Cowork / Code（含对照表）
│         ├─ 定义并压力测试问题假设（让 AI 唱反调）
│         ├─ 市场研究与竞争格局
│         │     竞争对手评估 / 市场研究 / 趋势分析
│         ├─ 规划设计客户探索（找谁聊·问什么·访后分析）
│         ├─ 客户触达与排期（Cowork 自动化）
│         ├─ 设计最终方案构想
│         └─ 用 Claude Code 搭轻量原型（只做核心交互→5人试用）
│
├─ 第4章 MVP 阶段 MVP Stage ★方法论核心
│   ├─ 目标：把已验证问题→可用产品；快不积债；投资持久上下文
│   ├─ 退出标准：产品—市场契合的真实证据（留存/营收/转介）
│   ├─ 挑战
│   │     ├─ 智能体式技术债（复利式增长）
│   │     ├─ 误信虚假 PMF（早期数据≠市场需要）
│   │     ├─ 零摩擦的范围蔓延
│   │     └─ 因经验不足而不安全（能跑≠安全）
│   └─ Claude 怎么帮
│         ├─ 构建前先定义架构（产出存为 CLAUDE.md）
│         ├─ 定义并执行 MVP 范围（范围文档 + 新增标准）
│         ├─ 用 Claude Code 构建（每次会话=执行已定决策）
│         ├─ 上线前安全审查（Claude Code Security）
│         ├─ 发布前建好度量框架（留存基准/激活/D7·D30）
│         ├─ 管理探索与反馈事务（保留真人解读）
│         ├─ 朝证据迭代而非朝「完整」（Sean Ellis / 努力测试）
│         └─ 证据要求时就转向（探索新客群 / 调价值主张 / 根本性改变）
│
├─ 第5章 发布阶段 Launch Stage ★方法论核心
│   ├─ 目标：早期牵引力→可重复可持续增长引擎；建运营系统释放注意力
│   ├─ 退出标准
│   │     ① 增长可重复·渠道驱动（CAC/LTV/回本）
│   │     ② 产品扛得住生产负载
│   │     ③ 运营无创始人瓶颈
│   ├─ 挑战
│   │     ├─ 技术债到期了
│   │     ├─ 创始人成为瓶颈
│   │     ├─ 安全合规不再可拖延
│   │     └─ 在准备好之前就扩张
│   └─ Claude 怎么帮（三形态协同放大）
│         ├─ 技术债复利前修复（Code 审计→Claude 排序）
│         ├─ 构建取代创始人注意力的系统（运营负载审计→分类）
│         ├─ 让安全合规成为产品工作流（SOC2/GDPR/HIPAA）
│         └─ 建立一直跳过的产品管理流程（冲刺/规格/bug分诊/周报）
│
├─ 第6章 规模化阶段 Scale Stage ★方法论核心
│   ├─ 目标：系统化增长；通过「积累的深度」建可防御护城河；扛外部审视
│   ├─ 退出标准：阈值事件（可持续盈利 / IPO就绪 / 被收购）
│   ├─ 挑战
│   │     ├─ 委派运营层（结构 + 心理双重挑战）
│   │     ├─ 扩展技术运营（成为可靠基础设施伙伴）
│   │     ├─ 扩展组织职能（招聘/薪酬/会计/法务）
│   │     └─ 构建 GTM 职能（有机增长撞天花板）
│   └─ Claude 怎么帮
│         ├─ 日常任务移交 Cowork（瓶颈图 / 一周不在测试）
│         ├─ 技术运营→企业级基础设施（文档/SLA/可观测性）
│         ├─ 构建真正的 GTM 职能（细分/信息架构/销售手册）
│         ├─ 领域知识→AI 上下文（对话/记忆/Skills 编码工作流）
│         ├─ 用户数据复利成可防御优势（数据飞轮/护城河叙事）
│         └─ 创造工作流锁定（集成深度/API/SDK→切换成本）
│
├─ 第7章 同样的工作，全新的规则（结论·升华）
│   ├─ 工作没变：找真问题→构建解决→扩成有分量的公司
│   ├─ 变的是路径：AI 把「季度」压缩成「周」
│   └─ 终极论断：瓶颈不再是「你能构建什么」，而是「你选择构建什么」
│
└─ 第8章 资源 Resources（附录）
    ├─ 用 Claude 构建（文档/最佳实践/CLAUDE.md/教程等 7 项）
    ├─ 创始人故事（YC三家 / GC AI / Carta / Anything / Cogent /
    │   Airtree / Duvo / Zingage / Kindora / Wordsmith）
    └─ 创业支持与机会（Startups Program / 社区 / 直播学习）
```

## 结构规律一句话总结

全书 = `背景(1-2章)` + `四阶段方法论(3-6章，每章固定四件套：目标→退出标准→挑战→Claude怎么帮+练习)` + `收尾(7章)` + `资源(8章)`。

其中 3–6 章是真正的「方法论主体」，彼此高度对称，可横向拉成一张「阶段 × 要素」矩阵表。
